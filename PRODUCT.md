# zerovibe — продукт

Платформа полного цикла для вайбкодеров: **пиши приложение на Go + HTMX с
Claude Code и деплой его к нам одной командой**. Мы арендуем VM, нарезаем на
контейнеры, раздаём по поддоменам с авто-TLS. Клиент не регистрируется у
облачного провайдера — платит нам (реселл инфры).

## Части

| Часть | Где | Что это |
|---|---|---|
| **Скелет** | `zerovibe/` корень (`cmd/`, `internal/`) | Эталонное приложение Go+HTMX+SQLite. Образец, по которому генерируются фичи. |
| **Плагин** | `zerovibe/plugin/` | Claude Code плагин: скиллы (конвенции, генерация фич, правила тестов) + деплой. То, что получает вайбкодер. |
| **Деплой (VM-стек)** | `zerovibe/deploy/` | Terraform (Timeweb VM) + VM-стек (Caddy+app) + скрипт доставки. Ручной флоу (Фаза 2). |
| **Оркестратор** | `platform/` (модуль `internal/usecase/deploy.go` и т.д.) | Управляющая плоскость переехала в **platform**: переиспользует её аккаунты вайбкодеров, проекты и auth. Проект = единица деплоя. |

> **Связь с platform:** аккаунты (админы/вайбкодеры), проекты и деплой-оркестрация
> живут в репозитории `platform`. zerovibe — это то, что отдаётся вайбкодеру
> (скелет + плагин) и инфра-обвязка VM. Платформа аутентифицирует вайбкодера,
> хранит его проекты и через модуль деплоя разворачивает их на VM.

## Поток вайбкодера

```
1. Старт со скелета (Go+HTMX+SQLite, чистая архитектура).
2. Claude Code + плагин zerovibe: "добавь раздел X" → код по эталону + тесты.
3. /zerovibe:deploy → оркестратор → контейнер на VM → https://app.zerovibe.ru
```

## Почему так

- **Стэк фиксирован** → не пишем движок-который-пишет-код (как в отложенном BaaS),
  а генерим обычное Go-приложение. Быстро и надёжно.
- **SQLite + очередь записи** → ноль внешних БД-зависимостей, дешёвая операционка,
  данные в РФ (152-ФЗ), бэкап = файл. Контейнер на VM снимает ограничение «один писатель».
- **Distroless-образ ~14 МБ** → дёшево хранить и быстро катить на VM.
- **Чистая архитектура везде** → и в скелете, и в оркестраторе; швы (порты)
  позволяют менять провайдера/деплоер, не трогая ядро.

## Статус (фазы)

- ✅ Фаза 1 — эталонный скелет (работает, тесты, Docker).
- ✅ Фаза 2 — деплой на 1 VM Timeweb вручную (Terraform + Caddy + deploy.sh; готово к apply).
- ✅ Фаза 3 — оркестратор: ядро размещения (заглушки провайдера/деплоера, тесты, API).
- ✅ Фаза 4 — Claude Code плагин (скиллы + деплой).
- 🔲 Реальные адаптеры оркестратора (Timeweb provider, SSH deployer), приём кода/образа, auth API.
- 🔲 Панель/CLI вайбкодера, биллинг, кастомные домены.

Отложено параллельно: BaaS-автобэк (репозиторий `platform`) — спрятан, не удалён.
