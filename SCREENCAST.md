# Сценарий скринкаста Zerovibe — «Трекер расходов от идеи до сайта»

Демонстрация: от пустой папки до живого сайта `*.zerovibe.ru` одной командой.
Хронометраж ~2–3 минуты. Аудитория — креаторы (не разработчики).
Главный посыл: **одна папка → один промпт → одна команда деплоя → живой сайт по HTTPS.**

---

## ЧАСТЬ A. Разовая подготовка машины (один раз, НЕ в кадре)

### A1. Получи/проверь account-ключ
Зайди в кабинет креатора → https://zerovibe.ru/login → раздел «Ключи» (`/app/keys`) →
«Создать ключ» → **скопируй секрет** (показывается один раз).
Если ключ уже сохранён — пропусти.

### A2. Установи/обнови плагин в Claude Code
Через меню (desktop): кнопка **+** у поля ввода → **Plugins** → **Add plugin** →
добавь маркетплейс `chudno/zerovibe` → установи плагин **zerovibe** → `/reload-plugins`.

Командами (одинаково в desktop и терминале):
```
/plugin marketplace add chudno/zerovibe
/plugin install zerovibe@zerovibe
```
ВАЖНО: если плагин уже стоял — обнови его (переустанови), чтобы подтянулась
версия 0.3.0+ с Franken UI и графиком.

### A3. Подключи ключ (один раз на машину)
```
/zerovibe:install
```
Вставь секрет → сохранится в `~/.zerovibe/config.json`.
**В кадре ключ не свети.**

### A4. Проверь, что версия свежая (критично для UI)
Создай пробную пустую папку, выполни `/zerovibe:init проверка`, убедись, что в
`template/internal/transport/web/static/` появились 6 файлов, включая
`frankenui-chart.iife.js`. Если файлов нет/мало — плагин старый, вернись к A2.
Пробную папку потом удали.

---

## ЧАСТЬ B. Репетиция (прогон без записи)

### B1. Чистая папка
```
mkdir ~/demo/wallet && cd ~/demo/wallet
```
Открой её в редакторе с Claude Code.

### B2. Заведи проект
```
/zerovibe:init wallet
```
Развернётся шаблон + создастся проект на платформе + `.env` с `ZEROVIBE_PROJECT`.

### B3. Один промпт — построй трекер (скопируй дословно)
> Преврати это в трекер личных расходов: трата = сумма, категория
> (еда/транспорт/развлечения/прочее), дата, комментарий. На главной — форма
> добавления, таблица трат с возможностью удалить, и сверху карточки: всего за
> месяц и круговой график расходов по категориям. Используй компоненты Franken UI
> и график `<uk-chart>`.

### B4. Посмотри локально
Скажи Claude «подними локально, покажи как запустить», или сам:
```
cd ~/demo/wallet && go run ./cmd/server
```
Открой `localhost:8080`, добавь 3–4 траты. Проверь: таблица, итоговая карточка,
круговой график, тёмная тема.
Если график не нарисовался — скажи Claude «график не отображается, проверь
подключение `<uk-chart>` и данные» (он сверится с `ui-kit/contexts/chart.md`).

### B5. Деплой (репетиционный)
```
/zerovibe:deploy
```
Дождись ссылки `wallet-xxx.zerovibe.ru`, открой — проверь HTTPS.
Засеки время сборки (первый build дольше — в монтаже ускоришь).

### B6. Если что-то сломалось — чини в репетиции, не в эфире.
Когда прогон прошёл гладко — записывай (можно тот же проект в новой чистой папке).

---

## ЧАСТЬ C. Запись (≈2–3 мин)

| Время | Кадр | Что делаешь / говоришь |
|---|---|---|
| 0:00–0:15 | Пустая папка | «Сделаю трекер расходов и опубликую в интернете — без сервера и DevOps» |
| 0:15–0:35 | `/zerovibe:init wallet` | Показать: развернулся шаблон с готовым UI, проект создан |
| 0:35–1:20 | Промпт (B3) | Claude пишет код по слоям, верстает на Franken UI, добавляет график (ускорить) |
| 1:20–1:45 | `localhost:8080` | Добавляешь траты → таблица + итог + круговой график крупно |
| 1:45–2:20 | `/zerovibe:deploy` | Сборка → деплой → появляется ссылка `*.zerovibe.ru` |
| 2:20–2:40 | Браузер, живой URL | Открываешь прод по HTTPS, добавляешь трату. Титр: «От идеи до сайта — минуты. Данные в России. zerovibe.ru» |

---

## ЧАСТЬ D. Технические настройки записи
- Крупный шрифт в редакторе/терминале, тёмная тема, разрешение 1080p+.
- Браузер заранее на вкладке `zerovibe.ru`, окно чистое (без лишних вкладок/закладок).
- Ключ нигде не должен попасть в кадр (ни `/app/keys`, ни `~/.zerovibe/config.json`,
  ни вывод `/zerovibe:install`).
- Деплой реальный, но долгое ожидание сборки ускорь в монтаже (ссылку покажи настоящую).
- Имя поддомена = имя проекта; выбери заранее, чтобы в кадре была красивая ссылка.

---

## Подводные камни (проверь до эфира)
1. Версия плагина — самый частый провал. Старый плагин = нет Franken UI и графика. Шаг A4 обязателен.
2. График — нужен `frankenui-chart.iife.js` (в 0.3.0 есть). Если агент не использует `<uk-chart>` — напомни про skill `ui-kit`.
3. Первый деплой медленный — Docker build на узле. Не паникуй в кадре.
4. Передеплой — если пишешь поверх того же проекта, поддомен и данные сохранятся (named volume).
5. Go должен быть установлен локально для шага B4 (`go run`). Если нет — пропусти локальный просмотр, показывай сразу прод после деплоя.

---

## Запасной промпт (если основной даёт слишком много для 45 сек)
> Сделай список трат: сумма, категория, дата. Форма добавления сверху, таблица
> снизу, и общая сумма за месяц. Оформи на Franken UI.

(Категории и график можно добавить вторым промптом уже в кадре — покажет итеративность.)
