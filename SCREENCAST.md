# Сценарий скринкаста Zerovibe — «Трекер расходов от макета до сайта»

Демонстрация: креатор рисует экран в Pencil → ИИ верстает по макету точь-в-точь →
публикация на живой сайт `*.zerovibe.app`. Хронометраж ~2–3 минуты.
Аудитория — креаторы (не разработчики).
Главный посыл: **нарисовал дизайн → ИИ собрал приложение по нему → опубликовал.**

---

## ЧАСТЬ A. Разовая подготовка машины (один раз, НЕ в кадре)

### A1. Получи/проверь account-ключ
Зайди в кабинет креатора → https://zerovibe.ru/login → раздел «Ключи» (`/app/keys`) →
«Создать ключ» → **скопируй секрет** (показывается один раз).
Если ключ уже сохранён — пропусти.

### A2. Установи/обнови плагин в Claude Code

Desktop-приложение НЕ поддерживает слэш-команды `/plugin` и добавление своего
маркетплейса через меню — используй настройки.

**Способ A (desktop и CLI):** добавь в `~/.claude/settings.json`:
```json
{
  "enabledPlugins": { "zerovibe@zerovibe": true },
  "extraKnownMarketplaces": {
    "zerovibe": {
      "source": { "source": "github", "repo": "chudno/zerovibe" },
      "autoUpdate": true
    }
  }
}
```
Затем **перезапусти приложение Claude**. `autoUpdate: true` сам подтянет свежую
версию (0.5.0+ — дизайн из Pencil, чистый CSS) — отдельно обновлять не нужно.

**Способ B (только CLI):**
```
/plugin marketplace add chudno/zerovibe
/plugin install zerovibe@zerovibe
/reload-plugins
```

### A3. Подключи ключ (один раз на машину)
```
/zerovibe:install
```
Вставь секрет → сохранится в `~/.zerovibe/config.json`.
**В кадре ключ не свети.**

### A4. Проверь, что версия свежая
Создай пробную пустую папку, выполни `/zerovibe:init проверка`, убедись, что:
- в `skills/` плагина есть `design/` (скил вёрстки из Pencil);
- в `template/internal/transport/web/static/` лежат только `app.css` и `htmx.min.js`
  (**не должно быть** `frankenui-*.js/css`).

Если это не так — плагин старый, вернись к A2. Пробную папку потом удали.

### A5. Подготовь Pencil и макет
- Pencil установлен, его MCP подключён к Claude Code (инструменты `mcp__pencil__*`
  доступны — проверь в сессии).
- Открой/нарисуй в Pencil экран трекера расходов (тёмная тема, фирменный фиолетовый
  акцент): шапка, карточки-итоги, форма добавления, таблица трат. Это и есть
  «источник истины» по виду — ИИ повторит его точь-в-точь.

## ЧАСТЬ B. Репетиция (прогон без записи)

### B1. Чистая папка
```
mkdir ~/demo/wallet && cd ~/demo/wallet
```
Открой её в редакторе с Claude Code. Pencil с макетом трекера — открыт рядом.

### B2. Заведи проект
```
/zerovibe:init wallet
```
Развернётся шаблон + создастся проект на платформе + `.env` с `ZEROVIBE_PROJECT`.

### B3. Один промпт — построй трекер по макету (скопируй дословно)
> Сделай трекер личных расходов: трата = сумма, категория
> (еда/транспорт/развлечения/прочее), дата, комментарий. На главной — форма
> добавления, таблица трат с возможностью удалить, и сверху карточки-итоги (всего
> за месяц и по категориям). **Свёрстай строго по макету, открытому в Pencil** —
> возьми оттуда цвета, отступы и компоновку.

Агент через skill `design` прочитает макет из Pencil (`get_variables`,
`snapshot_layout`, `batch_get`), положит токены в `static/app.css` и сверстает
экран по точным числам, затем оживит данные (HTMX) и бэкенд.

### B4. Посмотри локально
Скажи Claude «подними локально, покажи как запустить», или сам:
```
cd ~/demo/wallet && go run ./cmd/server
```
Открой `localhost:8080`, добавь 3–4 траты. Проверь главное: **экран совпадает с
макетом Pencil** — те же отступы, цвета, компоновка. Таблица и итоги работают.
Если что-то «уехало» от макета — скажи Claude «сверь с макетом в Pencil, поправь
отступы/цвета по нему» (он перечитает `snapshot_layout` и подгонит).

### B5. Деплой (репетиционный)
```
/zerovibe:deploy
```
Дождись ссылки `wallet-xxx.zerovibe.app`, открой — проверь HTTPS.
Засеки время сборки (первый build дольше — в монтаже ускоришь).

### B6. Если что-то сломалось — чини в репетиции, не в эфире.
Когда прогон прошёл гладко — записывай (можно тот же проект в новой чистой папке).

## ЧАСТЬ C. Запись (≈2–3 мин)

| Время | Кадр | Что делаешь / говоришь |
|---|---|---|
| 0:00–0:15 | Макет в Pencil | «Нарисовал трекер расходов. Сейчас ИИ соберёт по нему приложение и опубликует — без вёрстки руками и без сервера» |
| 0:15–0:35 | `/zerovibe:init wallet` | Показать: проект создан, папка готова |
| 0:35–1:25 | Промпт (B3) | Claude читает макет из Pencil и верстает по нему (видно вызовы `mcp__pencil__*`), пишет бэкенд (ускорить) |
| 1:25–1:50 | `localhost:8080` рядом с Pencil | Добавляешь траты → экран совпадает с макетом (покажи рядом: макет ↔ готовое приложение) |
| 1:50–2:20 | `/zerovibe:deploy` | Сборка → деплой → появляется ссылка `*.zerovibe.app` |
| 2:20–2:40 | Браузер, живой URL | Открываешь прод по HTTPS, добавляешь трату. Титр: «От макета до сайта — минуты. Данные в России. zerovibe.ru» |

Самый сильный кадр — **макет Pencil и готовое приложение рядом, один-в-один.**

---

## ЧАСТЬ D. Технические настройки записи
- Крупный шрифт в редакторе/терминале, тёмная тема, разрешение 1080p+.
- Браузер заранее на вкладке `zerovibe.ru`, окно чистое (без лишних вкладок/закладок).
- Pencil с макетом — на втором окне/экране, чтобы удобно показать «макет ↔ результат».
- Ключ нигде не должен попасть в кадр (ни `/app/keys`, ни `~/.zerovibe/config.json`,
  ни вывод `/zerovibe:install`).
- Деплой реальный, но долгое ожидание сборки ускорь в монтаже (ссылку покажи настоящую).
- Имя поддомена = имя проекта; выбери заранее, чтобы в кадре была красивая ссылка.

---

## Подводные камни (проверь до эфира)
1. Версия плагина — самый частый провал. Старый плагин = нет skill `design`, агент
   верстает «на глаз». Шаг A4 обязателен.
2. Pencil MCP — макет должен быть **открыт в Pencil**, инструменты `mcp__pencil__*`
   доступны в сессии. Если агент «не видит» макет — открой нужный экран в Pencil.
3. Первый деплой медленный — сборка на узле. Не паникуй в кадре.
4. Передеплой — если пишешь поверх того же проекта, поддомен и данные сохранятся.
5. Go должен быть установлен локально для шага B4 (`go run`). Если нет — пропусти
   локальный просмотр, показывай сразу прод после деплоя.

---

## Запасной промпт (если основной даёт слишком много для 45 сек)
> Сделай список трат: сумма, категория, дата. Форма сверху, таблица снизу, общая
> сумма за месяц. **Свёрстай по макету, открытому в Pencil.**

(Категории и итоги можно добавить вторым промптом уже в кадре — покажет, что ИИ
дорабатывает по тому же макету.)
