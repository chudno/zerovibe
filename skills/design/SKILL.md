# Вёрстка из макета Pencil

Внешний вид приложений Zerovibe **не придумывается агентом** — он берётся из
готового макета в Pencil. Pencil отдаёт точные числа (цвета, отступы, размеры,
шрифты, тексты), и твоя задача — перенести их в HTML-шаблон + `static/app.css`
**один в один**, а не «на глаз». Это убирает главную беду авто-вёрстки: рыхлые
отступы, цвета мимо, плоскую иерархию. Источник истины по виду — Pencil.

## Предусловие

Pencil уже установлен у пользователя, его MCP подключён к Claude Code, и нужный
макет **открыт в редакторе Pencil**. Инструменты `mcp__pencil__*` доступны.
Если инструменты не отвечают / макет не открыт — попроси пользователя открыть
нужный экран в Pencil и продолжай.

## Процедура (по порядку)

### 1. Найди экран в макете
`mcp__pencil__get_editor_state({ include_schema: true })` — посмотри top-level
frames (экраны) и список компонентов. Определи, какой frame соответствует экрану,
который нужно сверстать (по имени: «Admin Panel», «Dashboard», нужный раздел).
Если пользователь что-то выделил — бери выделение.

### 2. Забери токены дизайн-системы
`mcp__pencil__get_variables()` — отдаёт цвета/шрифты/числа как именованные
переменные (например `--brand-bg`, `--accent`, шрифты Geist). **Положи их в
`:root` в `static/app.css`** (заменяя дефолтные переменные шаблона). Это палитра
всего приложения — задаётся один раз.

### 3. Посмотри, как экран выглядит
`mcp__pencil__get_screenshot({ nodeId: "<frame-id>" })` — общий вид, чтобы понимать
композицию (что где, что главное). Это для ориентира, числа бери из шага 4.

### 4. Сними точную структуру
- `mcp__pencil__snapshot_layout({ parentId: "<frame-id>", maxDepth: 3 })` — даёт
  ПИКСЕЛЬНЫЕ координаты, размеры, вложенность каждого блока: ширина контейнера,
  отступы между секциями (gap), padding, высоты строк, ритм. **Это и есть отступы
  — переноси их как есть, не округляй «на вкус».**
- `mcp__pencil__batch_get({ nodeIds: [...], readDepth: 2, resolveVariables: true })`
  по нужным узлам — даёт `fill` (цвет), `cornerRadius`, `padding`, `gap`, `stroke`
  (бордер), `content` (текст), `fontFamily/fontSize/fontWeight`, иконки
  (`iconFontName`). Бери конкретные значения каждого элемента.

### 5. Перенеси в код
- Структуру → HTML в `templates/` (Go `html/template`), семантические теги.
- Все размеры/цвета/отступы → классы в `static/app.css`, опираясь на переменные из
  шага 2. Padding 20 → `padding: 20px`; gap 16 → `gap: 16px`; cornerRadius 12 →
  `border-radius: 12px`; fill `#111114` → `background: var(--surface)`. **Числа из
  Pencil, не выдуманные.**
- Данные/интерактив — по нашему HTMX-паттерну (см. skill `conventions`):
  `GET` отдаёт полную страницу, мутации (`POST`/`DELETE`) — HTML-фрагмент,
  `hx-post`/`hx-target`/`hx-swap`. Дизайн из Pencil статичен — оживляешь его ты.

### 6. Сверь результат
Сравни свою вёрстку со скриншотом из шага 3: отступы, выравнивание, цвета,
иерархия. Если «расползается» или плоско — вернись к `snapshot_layout` и поправь
числа. Не сдавай, пока не похоже на макет.

## Правила

- **Pencil — источник истины по виду.** Не добавляй «от себя» декор, не меняй
  палитру, не выдумывай отступы. Чего нет в макете — того нет.
- **Чистый CSS.** Никаких UI-библиотек/фреймворков (Tailwind, UIkit и т.п.) — только
  обычный CSS в `app.css` с переменными из Pencil.
- **Один источник палитры.** Токены из `get_variables` живут в `:root`, экраны
  ссылаются на переменные, не хардкодят цвета повторно.
- Если макета на нужный экран в Pencil нет — скажи пользователю, что нужен макет,
  и предложи сверстать по аналогии с уже существующими экранами проекта.
