# UI-кит Zerovibe — Franken UI

Веб-приложения Zerovibe оформляются библиотекой **Franken UI** — это компоненты
в стиле **shadcn/ui** на базе UIkit 3. Цель: каждое приложение креатора выглядит
современно и единообразно без ручной вёрстки CSS с нуля.

## Что уже подключено (НЕ трогай без причины)

Вся статика **локальная** (вшита в бинарь через `embed`, раздаётся с `/static/`) —
никаких CDN в рантайме (важно для работы в РФ и доступности). Подключено в
`internal/transport/web/templates/layout.html`:

- `frankenui-core.min.css` + `frankenui-utilities.min.css` — стили и utility-классы.
- `frankenui-core.iife.js` — компоненты (модалки, дропдауны, табы и т.п.).
- `frankenui-icon.iife.js` — иконки `<uk-icon>`.
- `frankenui-chart.iife.js` — графики `<uk-chart>` (ApexCharts под капотом).
- `htmx.min.js` — интерактивность.

Тема выставляется инлайн-скриптом в `<head>`: по умолчанию **тёмная** (`dark` на
`<html>`), тема `uk-theme-zinc`, скругления `uk-radii-md`, тени `uk-shadows-sm`.
`<body>` несёт `class="bg-background text-foreground"`.

## Правила вёрстки

1. **Используй компоненты Franken UI, не пиши свой CSS.** Для нового UI бери готовые
   классы `uk-*` и semantic-цвета (`bg-background`, `text-foreground`, `text-muted`,
   `uk-text-destructive`). Inline-`<style>` и кастомный CSS — только в крайнем случае.
2. **Сохраняй HTMX-паттерн** (см. skill `conventions`): `GET` → полная страница,
   мутации → HTML-фрагмент. Franken UI это не меняет — просто оформляет разметку.
3. **Иконки:** `<uk-icon icon="имя"></uk-icon>` (lucide-набор, напр. `trash`, `plus`).
4. **Тёмная тема — по умолчанию.** Не навязывай светлую; semantic-цвета сами
   адаптируются под `dark`.

## Композиция страницы (САМОЕ ВАЖНОЕ — не пропускай)

Franken UI даёт красивые компоненты, но НЕ собирает из них страницу. «Дешёвый»,
«сгенерированный ИИ» вид берётся не от компонентов, а от плохой композиции:
растянуто на весь экран, ровные одинаковые отступы, плоская иерархия. Это твоя
зона ответственности. Принципы (продуктовый UI — дашборды, формы, таблицы):

### Ширина и контейнер
- Контент НЕ должен растягиваться на весь широкий экран. Оборачивай страницу в
  `<div class="uk-container uk-container-xsmall">` (узкие формы/списки, ~640px) или
  `uk-container-small` (~960px). Без контейнера строки таблицы расползаются с
  пустыми провалами посередине — главный признак непродуманности.
- Текст прозы — не шире ~70 символов в строке.

### Ритм отступов (а не «везде одинаково»)
- Связанные элементы — близко; разные секции — далеко. Это и создаёт читаемость.
- Между крупными секциями страницы — крупный отступ (`uk-margin-large` /
  `uk-margin-xlarge`). Внутри секции между мелкими элементами — маленький
  (`uk-margin-small`). НЕ ставь один и тот же `uk-margin` везде — плоский ритм
  читается как черновик.
- Группируй: заголовок секции прижат к своему содержимому, а не висит в воздухе.

### Иерархия (squint-тест)
- Прищурься на макет: должно быть сразу видно, что главное, что второстепенное.
  Если всё одного веса — иерархии нет.
- Лейбл приглушай (`uk-text-muted`, мельче), значение выделяй (крупнее/жирнее).
  В строке «категория … сумма» сумма должна быть заметнее подписи.
- Заголовки — контрастной шкалой (`uk-h1`/`uk-h2`/`uk-h3`), не один размер на всё.

### Графики — В ЦВЕТ ТЕМЫ (частая ошибка)
- `<uk-chart>` по умолчанию рисует ЯРКО-СИНИМ (дефолт ApexCharts). На нашей
  тёмной zinc-теме синий выглядит инородно и «дёшево». ВСЕГДА задавай графику
  цвета явно через `colors` в опциях ApexCharts (`<script type="application/json">`).
- Для zinc-темы бери приглушённую нейтральную палитру, а не радугу. Пример набора
  для нескольких категорий (тёмная тема): `["#e4e4e7","#a1a1aa","#71717a","#52525b"]`
  (от светлого к тёмному по zinc-ramp). Одна серия — один светлый zinc (`#e4e4e7`).
- Подпиши `legend`/`labels` цветом текста темы (светлым), фон графика — прозрачный
  (`background: "transparent"`, `theme: { mode: "dark" }`). Деталь опций — `contexts/chart.md`.

### Состояния, без которых UI «недоделан»
- Пустое состояние (нет данных) — поясняющий текст, а не голый экран.
- У интерактивных элементов — hover/focus (Franken UI даёт их сам, не ломай).

## AI-slop: чего НЕ делать (бан)

Эти приёмы мгновенно выдают «сделано ИИ на отвал» — не используй:
- Цветная полоса-бордюр слева у карточек/алертов (`border-left: 4px …`).
- Градиентный текст (`background-clip: text`), декоративный glassmorphism.
- Одинаковые карточки-сетки «иконка + заголовок + текст» бесконечным рядом.
- Мелкие ВСЕ-КАПС «эйчбровки» над каждой секцией, нумерация `01 / 02 / 03` как
  украшение (номера только если это реальная последовательность шагов).
- Сверх-скругление (`border-radius: 32px+`) на карточках; карточки — 12–16px.
- Текст, вылезающий за контейнер на узких экранах (проверяй переносы).
- Маркетинговые штампы в текстах кнопок/заголовков. Кнопка = глагол + объект
  («Добавить трату», не «ОК»).

Проверка перед сдачей: «если бы кто-то глянул и сказал *это явно сделал ИИ* —
переделай». Чаще всего лечится композицией (ширина + ритм + иерархия), а не
заменой компонентов.

## Шпаргалка по частым компонентам

| Нужно | Классы / разметка |
|---|---|
| Карточка | `<div class="uk-card uk-card-body">…</div>` |
| Заголовок карточки | `<h3 class="uk-card-title">…</h3>` |
| Кнопка (основная) | `<button class="uk-btn uk-btn-primary">…</button>` |
| Кнопка (вторичная/призрак/опасная) | `uk-btn-secondary` / `uk-btn-ghost` / `uk-btn-destructive` |
| Текстовое поле | `<input class="uk-input">` |
| Многострочное | `<textarea class="uk-textarea">` |
| Селект | `<select class="uk-select">` |
| Чекбокс / радио | `uk-checkbox` / `uk-radio` |
| Группа полей | `<fieldset class="uk-fieldset space-y-4">` |
| Алерт | `<div class="uk-alert">` (+ `uk-alert-destructive`) |
| Бейдж | `<span class="uk-badge">` |
| Таблица | `<table class="uk-table">` |
| График | `<uk-chart>` (см. `contexts/chart.md`, `chart-line.md`, `chart-area.md`) |
| Контейнер страницы | `<div class="uk-container uk-container-small">` |

Эталон применения — в `internal/transport/web/templates/` (`layout.html`, `notes.html`,
`note.html`).

## Полная документация компонентов

Папка `contexts/` рядом с этим файлом — официальные context-файлы Franken UI
(~70 md, по одному на компонент: `card.md`, `form.md`, `table.md`, `modal.md`,
`dropdown.md`, `tab.md`, `select.md` и т.д., плюс `theming.md`, `installation.md`,
`icon-library.md`). Когда нужен компонент, которого нет в шпаргалке, или его модификаторы —
**прочитай соответствующий `contexts/<имя>.md`** перед вёрсткой, не угадывай классы.
