# Локальный запуск (посмотреть перед деплоем)

Пользователь — НЕ программист (см. skill `conventions`). Все технические действия делает
АГЕНТ САМ: поднимает сервер, создаёт первого администратора, проверяет, что всё ожило.
Человеку ты отдаёшь только **ссылку и (если нужно) логин/пароль** простыми словами — он
открывает ссылку в браузере и смотрит.

> Зачем: «сначала посмотри, потом в прод». Дай человеку увидеть результат вживую ДО
> деплоя — на правки уходит секунды, а в проде уже на глазах у его пользователей.

## Когда предлагать (без напоминания пользователя)

После любой ЗАМЕТНОЙ работы — добавил фичу/раздел, сверстал экран, поменял дизайн,
поправил поведение — и ПЕРЕД тем как деплоить. Просто подними и дай ссылку: «Поднял
локально, посмотри — http://localhost:8080. Нравится — выкатываю в прод». Не спрашивай
разрешения поднять; спрашивай только готов ли деплоить ПОСЛЕ просмотра.

## Шаги (агент делает сам)

### 1. Поднять сервер в фоне
Из папки приложения (где `cmd/server`). Запускай в ФОНЕ, чтобы продолжать работу и не
блокировать сессию. Своя временная БД-файл, свой `SETUP_TOKEN`, чтобы создать админа:

```sh
SETUP_TOKEN=local-setup DB_PATH="file:./zerovibe-local.db" ADDR=":8080" \
  SECURE_COOKIE=false go run ./cmd/server
```
(`make run` тоже поднимает на `:8080`, но через env удобнее задать SETUP_TOKEN и порт.)

Дождись, пока поднимется: `curl -fsS http://localhost:8080/healthz` отдаёт `ok`.
Если порт занят — возьми соседний (`:8081`…) и используй его в ссылке.

### 2. Создать первого администратора (если в приложении есть вход)
Вайбкодер НЕ делает запросы — дёрни `/setup` САМ (создаёт первого админа один раз):

```sh
curl -s -X POST http://localhost:8080/setup \
  -d 'email=<email>&password=<пароль>&token=local-setup'
```
Email/пароль — те, что пользователь хочет (спроси обычными словами, если не знаешь:
«придумай почту и пароль для входа»). Если входа в приложении нет — пропусти этот шаг.

### 3. (Опционально) Насеять немного данных
Если без данных смотреть нечего (пустые списки) — создай 2-3 примера через формы/админку
приложения, чтобы человеку было на что посмотреть. Не обязательно.

### 4. Дать пользователю ссылку
Простыми словами, без жаргона:
> Поднял локально — посмотри: **http://localhost:8080**
> Вход: `<email>` / `<пароль>` (если есть админка — `http://localhost:8080/admin`)
> Как глянешь — скажи, и я выкачу в прод (или поправлю, что не так).

## Грабли

- **Запускай в фоне** — иначе сессия блокируется. Останови процесс, когда пользователь
  посмотрел и решил (`pkill -f 'go run ./cmd/server'` или по порту).
- **SQLite-файл локальный** (`./zerovibe-local.db`) — не коммить его (он в `.gitignore`
  по маске; если нет — не добавляй в git). При повторном запуске админ уже есть — `/setup`
  второй раз вернёт ошибку, это норма (можно удалить файл БД для чистого старта).
- **Платформенные фичи без ключей** — локально нет `PLATFORM_API_KEY`: письма пишутся
  в лог (не уходят), файлы складываются в каталог `uploads/` вместо S3. Это нормально
  для просмотра — флоу работает, см. грейсфул-фолбэки в `platformmail`/`platformfiles`.
- **`SECURE_COOKIE=false`** обязательно локально (без него cookie сессии не встанет по http).
- **Порт занят** — не убивай чужой процесс молча; возьми другой порт и дай ссылку на него.
- Останови предыдущий локальный сервер перед новым запуском (rate-limit на вход и порт).

## После просмотра

Пользователь посмотрел → готов к проду → запускай skill `deploy`. Перед этим погаси
локальный сервер. Если просит правки — поправь и подними снова (тот же цикл).

Аргумент пользователя: $ARGUMENTS
