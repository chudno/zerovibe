# Новый проект на Zerovibe

Разворачивает в текущей папке рабочий шаблон, создаёт проект на платформе и
привязывает папку к нему. После этого можно сразу писать фичи и публиковать.

## Предусловие

Нужен сохранённый ключ (`~/.zerovibe/config.json`). Нет → сначала `/zerovibe:install`.

## Что нужно сделать

**1. Развернуть шаблон в текущую папку — только если она пуста.** Эталонный шаблон
лежит внутри плагина по пути `${CLAUDE_SKILL_DIR}/../../template`. Если в папке уже
есть код (маркеры: `go.mod` или `Dockerfile`) — НЕ затирай его, просто привяжи
существующий код. Иначе скопируй шаблон. После — по желанию пользователя `git init`.

**2. Создать проект на платформе:**

> `POST {api}/dev/projects` с заголовком `X-API-Key: <ключ>`, тело `{"name": "<имя>"}`.
> ID проекта — в ответе по пути `project.id` (учти вложенность).

Имя проекта: из аргумента пользователя; если пусто — предложи имя текущей папки и
подтверди.

**3. Привязать папку** — записать в `.env` проекта `ZEROVIBE_PROJECT=<id>` и
`ZEROVIBE_API=<api>` (идемпотентно: если строки уже есть — обнови). Ключ в `.env`
НЕ пишем — он глобальный, в `~/.zerovibe/config.json`. Убедись, что `.env` в
`.gitignore` (его не коммитим).

**4. Подвести итог простыми словами** (креатор не должен лезть в доки). Примерно:

> Готово! Проект «<имя>» создан, папка привязана. Дальше просто говори обычными
> словами, что нужно в приложении — например «добавь раздел задач» или «сделай форму
> входа». Когда захочешь опубликовать — скажи «задеплой», и приложение появится по
> адресу `<поддомен>.zerovibe.ru`.

## Замечания

- Все вызовы платформы — заголовком **`X-API-Key`** (не Bearer).
- Один проект = одна папка. Для другого приложения — отдельная папка + свой `/zerovibe:init`.

Аргумент пользователя (имя проекта): $ARGUMENTS
