# Подключение к Zerovibe

Разовая настройка машины: сохранить ключ креатора и поставить инструмент сборки,
чтобы дальше можно было создавать проекты, проверять их локально и публиковать.

## Что нужно сделать

**1. Получить ключ.** Если пользователь не дал его сразу — попроси создать в кабинете:
`https://zerovibe.ru` → раздел «Ключи» (`/app/keys`) → создать → скопировать секрет
(показывается один раз).

**2. Сохранить ключ** в `~/.zerovibe/config.json` — это глобальный конфиг машины
(один ключ на все проекты креатора). Формат и доступ:

```json
{ "api": "https://zerovibe.ru/v1", "key": "<секрет>" }
```

- `api` по умолчанию `https://zerovibe.ru/v1` (менять не нужно).
- Права строго `0600` (файл) и `0700` (каталог) — это секрет, доступ только владельцу.
- Этот файл НЕ коммитить.

**3. Проверить ключ** реальным вызовом — лёгкий листинг, данные не меняет:

> `GET {api}/dev/projects` с заголовком `X-API-Key: <ключ>`
> `200` → ключ рабочий. `401` → неверный/отозван, попроси создать новый.

Все вызовы к платформе авторизуются заголовком **`X-API-Key`** (не Bearer).

**4. Поставить инструмент сборки (Go ≥ 1.25)** — нужен для локальной проверки
приложения (тесты + предпросмотр перед публикацией). Проверь, есть ли подходящая
версия; если нет — установи любым удобным для системы способом.

Грабля: пакетные менеджеры на старых дистрибутивах нередко дают устаревший Go
(< 1.25). Если так — возьми актуальную версию с официального источника
(`https://go.dev/dl/`) и убедись, что она попала в `PATH`. Версия 1.25 обязательна —
этого требует шаблон.

Если поставить Go не удалось (нет прав/интернета) — это НЕ блокирует публикацию
(платформа собирает приложение сама), но локальные тесты и предпросмотр будут
недоступны. Честно скажи об этом пользователю.

## Как объяснять пользователю

Креатор — не программист. Не вали на него техническими деталями: не «установил Go
1.25 и сохранил API-ключ в config.json», а «подключил тебя к платформе и поставил
всё нужное, чтобы проверять приложение прямо на твоём компьютере перед публикацией».

## Замечания

- Ключ — глобальный (на все проекты). Конкретный проект привязывается отдельно
  командой `/zerovibe:init`.

Аргумент пользователя (может содержать ключ): $ARGUMENTS
