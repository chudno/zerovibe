# Подключение к Zerovibe (установка ключа)

Сохраняет API-ключ вайбкодера в глобальный конфиг `~/.zerovibe/config.json`
(один раз на машину). Дальше команды `/zerovibe:new` и деплой берут ключ оттуда —
в каждом проекте вводить не нужно.

## Где взять ключ

Ключ создаётся в веб-кабинете вайбкодера:
1. Зайти на https://zerovibe.ru → войти.
2. Раздел **«Ключи»** (`/app/keys`) → «Создать ключ».
3. Скопировать показанный секрет (он показывается один раз).

Если у пользователя нет ключа — попроси его создать по шагам выше и вставить сюда.

## Что делает skill

1. **Запроси у пользователя ключ** (если не передан аргументом). Пользователь
   вставляет секрет ключа в чат.
2. **Сохрани конфиг** в `~/.zerovibe/config.json` с правами 0600 (только владелец):

   ```sh
   mkdir -p ~/.zerovibe && chmod 700 ~/.zerovibe
   cat > ~/.zerovibe/config.json <<EOF
   {
     "api": "https://zerovibe.ru/v1",
     "key": "ВСТАВЛЕННЫЙ_КЛЮЧ"
   }
   EOF
   chmod 600 ~/.zerovibe/config.json
   ```

   `api` по умолчанию `https://zerovibe.ru/v1` — менять не нужно.

3. **Проверь ключ** реальным вызовом (листинг проектов — лёгкий, не меняет данные):

   ```sh
   KEY=$(python3 -c 'import json;print(json.load(open(__import__("os").path.expanduser("~/.zerovibe/config.json")))["key"])')
   API=$(python3 -c 'import json;print(json.load(open(__import__("os").path.expanduser("~/.zerovibe/config.json")))["api"])')
   curl -fsS "$API/dev/projects" -H "X-API-Key: $KEY" -o /dev/null -w "auth check: %{http_code}\n"
   ```

   - `200` → ключ рабочий, подтверди пользователю: «Готово, ключ сохранён».
   - `401` → ключ неверный/отозван. Сообщи и попроси создать новый в кабинете.

## Замечания

- Ключ хранится в plaintext в `~/.zerovibe/config.json` (как `~/.aws/credentials`).
  Права 0600 — только владелец. Не коммить этот файл.
- Ключ — глобальный (на все проекты вайбкодера). Конкретный проект привязывается
  отдельно командой `/zerovibe:new` (пишет ZEROVIBE_PROJECT в `.env` проекта).

Аргумент пользователя (может содержать ключ): $ARGUMENTS
